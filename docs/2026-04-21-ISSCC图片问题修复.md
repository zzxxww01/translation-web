# ISSCC 2026 图片显示问题修复报告

## 问题描述

导出的 ISSCC 2026 中文 Markdown 文件中，大量图片无法显示。

## 根本原因

**Substack CDN URL 包含动态占位符**：
```
https://substackcdn.com/image/fetch/$s_!8SFa!,w_1456,c_limit,.../https%3A%2F%2Fsubstack-post-media.s3.amazonaws.com%2F...
                                      ^^^^^^^^
                                      占位符，在浏览器中由 JS 动态替换
```

这些占位符 `$s_!xxxx!` 在原始 HTML 中会被 JavaScript 动态替换成实际的尺寸参数，但在导出的 Markdown 中没有被处理，导致图片链接失效。

---

## 技术分析

### 旧版正则表达式（有缺陷）

```python
# src/html2md/images.py:22
_SUBSTACK_CDN_PATTERN = re.compile(
    r"https?://substackcdn\.com/image/fetch/[^/]*/(?P<encoded_url>https?%3A%2F%2F.+)"
)
```

**问题**：
- `[^/]*` 只匹配"非斜杠字符"
- 无法处理包含逗号分隔的多个参数
- 无法处理占位符 `$s_!xxxx!`

### 新版正则表达式（通用方案）

```python
# src/html2md/images.py:22-24
_SUBSTACK_CDN_PATTERN = re.compile(
    r"https?://substackcdn\.com/image/fetch/.+/(https?%3A%2F%2F.+)"
)
```

**改进**：
- `.+/` 贪婪匹配所有参数（包括占位符、逗号、斜杠等）
- 直接提取最后一个 URL 编码的 `https://` 地址
- 通用性强，适配未来可能的 URL 格式变化


### 方案 2：手动修复导出文件（临时方案）

使用正则替换修复已导出的 Markdown 文件：

```python
import re
from urllib.parse import unquote

def fix_substack_images(markdown_content):
    pattern = r'!\[(.*?)\]\(https://substackcdn\.com/image/fetch/.+/(https?%3A%2F%2F[^)]+)\)'
    
    def replace(match):
        alt = match.group(1)
        encoded_url = match.group(2)
        decoded_url = unquote(encoded_url)
        return f'![{alt}]({decoded_url})'
    
    return re.sub(pattern, replace, markdown_content)

# 使用
with open('ISSCC_2026_zh.md', 'r', encoding='utf-8') as f:
    content = f.read()

fixed_content = fix_substack_images(content)

with open('ISSCC_2026_zh_fixed.md', 'w', encoding='utf-8') as f:
    f.write(fixed_content)
```
