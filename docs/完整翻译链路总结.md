# 长文翻译完整链路总结

## 一、整体架构

### 1.1 核心组件

```
BatchTranslationService (批量翻译服务)
    ├─ DeepAnalyzer (深度分析器)
    ├─ FourStepTranslator (四步法翻译器)
    ├─ ConsistencyReviewer (一致性审查器)
    ├─ LayeredContextManager (分层上下文管理器)
    └─ LLMProvider (模型提供者)
```

### 1.2 阶段化模型配置

```yaml
# config/translation_models.yaml
phase0_prescan: grok-4-20-non-reasoning (temp=0.3)  # 深度分析
phase1_draft: deepseek-v3 (temp=0.3)                # 初译
phase2_refine: deepseek-v3 (temp=0.3)               # 精修
phase3_review: deepseek-v3 (temp=0.3)               # 审查
```

## 二、完整翻译流程

### Phase 0: 深度分析（全文级）

**执行者**: `DeepAnalyzer`  
**模型**: Grok-4 (non-reasoning, temp=0.3)  
**频率**: 每篇文章 1 次  
**成本**: ~$0.02-0.05/文章

#### 输入
- 完整原文（所有章节）
- 文章元信息（标题、作者、来源）

#### 输出
```json
{
  "article_type": "technical_deep_dive",
  "target_audience": "AI engineers, researchers",
  "key_themes": ["AI infrastructure", "model serving"],
  "tone": "professional, technical",
  "terminology_density": "high",
  "sections": [
    {
      "section_id": "ch1",
      "title": "Introduction",
      "role": "background",
      "key_concepts": ["CPO", "scale-up"],
      "translation_challenges": ["technical terms"]
    }
  ]
}
```

#### 作用
- 理解文章整体结构和主题
- 识别关键术语和翻译挑战
- 为后续章节翻译提供全局上下文

---

### Phase 1: 初译（章节级）

**执行者**: `FourStepTranslator`  
**模型**: DeepSeek v3 (temp=0.3)  
**频率**: 每个章节 1 次  
**成本**: ~$0.0012/章节（3段）

#### Step 0: 章节预扫描（可选）

**模型**: Flash (快速模型)  
**作用**: 提取新术语，检测术语冲突

```json
{
  "new_terms": [
    {
      "term": "co-packaged optics",
      "translation": "共封装光学",
      "confidence": 0.95
    }
  ],
  "conflicts": [
    {
      "term": "scale-up",
      "existing": "扩展",
      "suggested": "纵向扩展",
      "severity": "high"
    }
  ]
}
```

#### Step 1: 理解章节

**输入**:
- 当前章节内容
- Phase 0 全文分析结果
- 前后章节标题和摘要

**输出**:
```json
{
  "section_role": "introduces CPO technology",
  "key_points": ["bandwidth bottleneck", "optical solution"],
  "dependencies": ["ch2: technical details"],
  "translation_focus": ["maintain technical precision"]
}
```

#### Step 2: 批量初译

**模式**: 
- 小章节（≤8段）: 一次性翻译
- 大章节（>8段）: 分批翻译（每批 8 段）

**输入**:
```
原文段落 1-8
+ 章节理解
+ 术语表
+ 翻译指南
+ 全文上下文
```

**输出**:
```json
{
  "translations": [
    {
      "paragraph_id": "p1",
      "source": "Co-packaged optics (CPO) represents...",
      "translation": "共封装光学（CPO）代表了...",
      "format_tokens": ["[[[LINK_1|...]]]"]
    }
  ]
}
```

**成本**: $0.0012/章节（3段）

---

### Phase 2: 精修（章节级）

**执行者**: `FourStepTranslator`  
**模型**: DeepSeek v3 (temp=0.3)  
**频率**: 根据质量评分智能触发  
**成本**: $0.0010-0.0026/章节（优化后）

#### Step 3: 反思（必执行）

**作用**: Phase 2 唯一入口，100% 覆盖所有段落

**输入**:
```
原文段落 1-N
+ 译文段落 1-N
+ 术语表
+ 翻译指南
```

**输出**: 6 维度评分 + 问题列表

```json
{
  "scores": {
    "overall": 8.6,
    "terminology": 9.0,      // 术语准确性
    "accuracy": 9.1,         // 信息准确性
    "fluency": 8.2,          // 流畅度
    "conciseness": 8.0,      // 简洁性
    "consistency": 8.5,      // 一致性
    "logic": 8.8             // 逻辑性
  },
  "statistics": {
    "critical_issues": 0,    // P0 致命错误
    "high_issues": 2,        // P1 严重问题
    "medium_issues": 3,      // P2 中等问题
    "low_issues": 1          // P2 轻微问题
  },
  "is_excellent": false,
  "issues": [
    {
      "paragraph_index": 2,
      "priority": "P1",
      "issue_type": "terminology",
      "severity": "high",
      "original_text": "scale-up",
      "description": "术语翻译不准确",
      "why_it_matters": "技术概念混淆",
      "suggestion": "改为'纵向扩展'"
    }
  ]
}
```

**成本**: $0.0011/章节（3段）

#### 评分标准（方案 C）

| 分数区间 | 等级 | 后续动作 |
|---------|------|---------|
| 9.5-10.0 | 完美 | 结束 Phase 2 ✓ |
| 9.0-9.4 | 优秀 | 结束 Phase 2 ✓ |
| 8.5-8.9 | 良好 | Step 4（只修 P0/P1） |
| 8.0-8.4 | 合格 | Step 4（修 P0/P1/P2） |
| <8.0 | 需改进 | Step 4（修所有问题） |

#### Step 4: 精修（条件触发）

**触发条件**:
```python
if overall_score < 9.0 or has_critical_issues:
    execute_step4()
```

**问题过滤策略**:
```python
if overall_score >= 8.5:
    # 只修 P0/P1 高优先级问题
    issues_to_fix = [P0, P1]
elif overall_score >= 8.0:
    # 修 P0/P1/P2
    issues_to_fix = [P0, P1, P2]
else:
    # 修所有问题
    issues_to_fix = [P0, P1, P2, P3]
```

**批量精修**: 一次 API 调用修正所有问题

**输入**:
```
原文段落 1-N
+ 当前译文 1-N
+ 问题列表（已过滤）
```

**输出**:
```json
{
  "refined_translations": [
    {
      "paragraph_id": "p2",
      "original": "scale-up 扩展",
      "refined": "纵向扩展",
      "changes": "修正术语翻译"
    }
  ]
}
```

**成本**: $0.0007/章节（3段，假设 2 个问题）

#### Step 5: 风格润色（条件触发）

**触发条件**:
```python
if fluency_score < 8.5 or conciseness_score < 8.5:
    execute_step5()
```

**批量润色**: 一次 API 调用润色所有段落

**输入**:
```
原文段落 1-N
+ 当前译文 1-N
```

**输出**:
```json
{
  "polished_translations": [
    {
      "paragraph_id": "p1",
      "before": "共封装光学（CPO）代表了一种新的方法...",
      "after": "共封装光学（CPO）是一种新方法...",
      "improvement": "简化表达，提升简洁性"
    }
  ]
}
```

**成本**: $0.0009/章节（3段）

#### Phase 2 成本总结

| 场景 | Step 3 | Step 4 | Step 5 | 总计 | API 调用 |
|------|--------|--------|--------|------|---------|
| 优秀（80%） | $0.0011 | - | - | $0.0011 | 1 次 |
| 良好（15%） | $0.0011 | $0.0007 | - | $0.0018 | 2 次 |
| 需润色（5%） | $0.0011 | $0.0007 | $0.0009 | $0.0027 | 3 次 |
| **加权平均** | - | - | - | **$0.0013** | **1.25 次** |

**优化效果**:
- API 调用: 7 次 → 1-3 次（减少 57-86%）
- Phase 2 成本: $0.0026 → $0.0013（降低 50%）
- Phase 2/Phase 1 比例: 2.2x → 1.08x

### Phase 3: 一致性审查（全文级）

**执行者**: `ConsistencyReviewer`  
**模型**: DeepSeek v3 (temp=0.3)  
**频率**: 每篇文章 1 次  
**成本**: ~$0.01-0.03/文章

#### 输入
- 所有章节的译文
- 术语表
- Phase 0 分析结果

#### 检查项
1. **术语一致性**: 同一术语在全文中的翻译是否统一
2. **风格一致性**: 语气、表达方式是否一致
3. **逻辑连贯性**: 章节之间的衔接是否自然
4. **格式一致性**: 标题、列表、代码块格式是否统一

#### 输出
```json
{
  "consistency_score": 9.2,
  "issues": [
    {
      "type": "terminology_inconsistency",
      "term": "scale-up",
      "locations": ["ch1:p2", "ch3:p5"],
      "translations": ["扩展", "纵向扩展"],
      "suggestion": "统一使用'纵向扩展'"
    }
  ]
}
```

## 三、关键优化机制

### 3.1 阶段化模型支持

**目标**: 不同阶段使用不同模型，平衡质量和成本

```python
# BatchTranslationService
def _get_provider_for_phase(self, phase: str) -> LLMProvider:
    """根据阶段返回对应的 provider"""
    model_config = self.model_config.get_model_for_phase(
        phase, self.user_model_override
    )
    return self.provider_factory.create_provider(model_config)

# FourStepTranslator
phase1_provider = self.get_provider_for_phase("phase1_draft")
phase2_provider = self.get_provider_for_phase("phase2_refine")
```

**效果**:
- Phase 1 使用 DeepSeek v3: 快速、低成本
- Phase 2 使用 DeepSeek v3: 保持质量
- 用户可覆盖任意阶段的模型

### 3.2 智能触发机制

**Phase 2 智能跳过**:
```python
# Step 3 反思后
if overall_score >= 9.0 and no_critical_issues:
    # 质量优秀，跳过 Step 4 和 Step 5
    return translations
```

**Step 4 问题过滤**:
```python
# 根据 overall_score 决定修复哪些问题
if overall_score >= 8.5:
    issues_to_fix = [P0, P1]  # 只修高优先级
else:
    issues_to_fix = [P0, P1, P2]  # 修更多问题
```

**Step 5 条件触发**:
```python
# 只在流畅度或简洁性不足时触发
if fluency < 8.5 or conciseness < 8.5:
    execute_polish()
```

### 3.3 批量处理优化

**Step 3 反思**: 批量分析所有段落
- 输入: N 个段落
- 输出: N 个段落的评分 + 问题列表
- API 调用: 1 次

**Step 4 精修**: 批量修正所有问题
- 输入: N 个段落 + M 个问题
- 输出: N 个修正后的段落
- API 调用: 1 次

**Step 5 润色**: 批量润色所有段落
- 输入: N 个段落
- 输出: N 个润色后的段落
- API 调用: 1 次

### 3.4 6 维度评分体系

| 维度 | 权重 | 作用 |
|------|------|------|
| terminology | 20% | 识别术语错误 |
| accuracy | 25% | 识别信息失真 |
| fluency | 20% | 识别翻译腔 |
| conciseness | 15% | 识别冗余 |
| consistency | 10% | 识别不一致 |
| logic | 10% | 识别逻辑问题 |

**Overall 计算**:
```python
overall = (
    terminology * 0.20 +
    accuracy * 0.25 +
    fluency * 0.20 +
    conciseness * 0.15 +
    consistency * 0.10 +
    logic * 0.10
)
```

## 五、质量保证机制

### 5.1 问题优先级

| 优先级 | 定义 | 示例 | 处理策略 |
|--------|------|------|---------|
| P0 | 致命错误 | 事实错误、数字错误、术语严重错误 | 必须修复 |
| P1 | 严重偏离 | 术语不一致、表达不清、语气不当 | 强烈建议修复 |
| P2 | 可选修复 | 表达略显生硬、可以更简洁 | 根据 overall_score 决定 |

### 5.2 质量门槛

```python
# Step 3 反思后
if overall_score >= 9.0 and no_P0_P1_issues:
    # 优秀质量，直接通过
    return translations

if overall_score >= 8.5:
    # 良好质量，只修 P0/P1
    refine_issues([P0, P1])

if overall_score >= 8.0:
    # 合格质量，修 P0/P1/P2
    refine_issues([P0, P1, P2])

if overall_score < 8.0:
    # 质量不足，考虑重译
    retry_or_refine_all()
```

### 5.3 术语一致性保证

1. **Phase 0**: 提取全文关键术语
2. **Step 0**: 章节预扫描，检测术语冲突
3. **Step 2**: 初译时注入术语表
4. **Step 3**: 反思时检查术语一致性
5. **Phase 3**: 全文审查术语统一性

### 6.2 参数调优

**style_polish_threshold**:
- 默认: 8.0（平衡质量和成本）
- 高质量: 8.5（更频繁润色）
- 低成本: 7.5（减少润色）
- 关闭: 0（完全跳过 Step 5）

**paragraph_threshold**:
- 默认: 8（超过 8 段分批翻译）
- 小章节: 12（减少分批次数）
- 大章节: 6（更细粒度控制）

### 6.3 成本优化策略

1. **Phase 1 使用 DeepSeek v3**: 节省 90% 成本
2. **启用智能跳过**: 80% 章节跳过 Step 4/5
3. **批量处理**: Step 4/5 使用批量 API
4. **提高评分标准**: 减少不必要的精修

**预期效果**:
- 总成本降低 60-70%
- 质量保持或提升
- 翻译速度提升 2-3 倍

## 八、总结

### 8.1 核心优势

1. **阶段化模型**: 不同阶段使用最优模型，平衡质量和成本
2. **智能触发**: 基于多维度评分，精准决定后续步骤
3. **批量处理**: Step 3/4/5 全部批量化，减少 API 调用
4. **6 维度评分**: 全面评估翻译质量，精准识别问题
5. **问题优先级**: P0/P1/P2 分级，智能过滤修复范围

### 8.2 关键数据

- **Phase 2 成本**: 从 2.2x Phase 1 降至 1.08x
- **API 调用**: 从 7 次降至 1-3 次
- **总成本**: 降低 60-70%
- **质量**: 保持或提升（6 维度评分）

### 8.3 适用场景

✅ **适合**:
- 技术文章、博客、文档
- 需要高质量翻译
- 成本敏感项目
- 大批量翻译任务

❌ **不适合**:
- 极短文本（<100 字）
- 实时翻译需求
- 非结构化内容
