# 2026-04-20 修复总结

## 问题背景

用户报告两个翻译功能失败：
1. **帖子翻译**：报错 `Request validation failed`
2. **长文翻译**：报错 `'list' object has no attribute 'items'`

## 根本原因分析

### 问题 1：API 路由参数缺失

**原因**：`generate_with_fallback()` 函数签名要求 `task_type` 参数（默认值 `"post"`），但多个 API 端点调用时未传递此参数，导致模型路由失败。

**影响范围**：5 个 API 端点
- `/translate/post` (translate_posts.py:61)
- `/translate/post/optimize` (translate_posts.py:101)
- `/generate/title` (translate_posts.py:131)
- `/tools/email-reply` (tools_email.py:75)
- `/tools/translate` (tools_translate.py:47)

### 问题 2：Pydantic 模型序列化错误

**原因**：`_normalize_artifact_payload()` 在处理嵌套 Pydantic 模型时，`model_dump()` 返回的字典可能包含未转换的子模型（如 `ArticleStyle`、`TranslationChallenge`），导致后续调用 `.items()` 时失败。

**影响范围**：长文翻译的四步法流程（batch_translation_service.py:301）

## 修复方案

### 1. API 调用修复

为所有缺失的调用添加 `task_type` 参数：

```python
# 修复前
generate_with_fallback(prompt, model=body.model)

# 修复后
generate_with_fallback(prompt, task_type="post", model=body.model)
```

**修复文件**：
- `src/api/routers/translate_posts.py` (3 处)
- `src/api/routers/tools_email.py` (1 处)
- `src/api/routers/tools_translate.py` (1 处)

### 2. Pydantic 序列化修复

递归处理 `model_dump()` 的结果，确保所有嵌套模型都被正确转换：

```python
# 修复前
if hasattr(payload, "model_dump"):
    return payload.model_dump(mode="json")

# 修复后
if hasattr(payload, "model_dump"):
    return self._normalize_artifact_payload(payload.model_dump(mode="json"))
```

**修复文件**：
- `src/services/batch_translation_service.py:301`

### 3. 代理配置优化

**变更内容**：
- Gemini 和 VectorEngine 默认改为 `proxy_mode: disabled`
- 支持通过 YAML 独立控制每个 Provider 的代理策略
- VectorEngine 添加备用 Key 支持

**修改文件**：
- `config/llm_providers.yaml`
- `.env`

## 技术要点

### 故障转移机制

当前系统的故障转移顺序：
1. **同组内 Key 轮换**：主 Key → 备用 Key
2. **同组内模型切换**：当前模型 → 同组其他模型
3. **跨组模型切换**：当前组 → 其他组（按 `group_priority` 排序）

### 任务类型映射

| 任务类型 | 默认模型 | 用途 |
|---------|---------|------|
| `longform` | `deepseek-v3` | 长文翻译 |
| `post` | `deepseek-v3` | 帖子翻译 |
| `analysis` | `deepseek-v3` | 文本分析 |
| `title` | `deepseek-v3` | 标题生成 |
| `metadata` | `deepseek-v3` | 元数据提取 |
| `slack` | `gemini-pro` | Slack 辅助 |

### 代理配置模式

| 模式 | 说明 | 适用场景 |
|-----|------|---------|
| `disabled` | 无需代理 | 海外服务器、国内可直连的服务 |
| `required` | 必须代理 | 受限网络、需要代理访问的服务 |
