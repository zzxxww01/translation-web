# 模型配置说明

## 配置架构

系统支持**两种Provider类型**，每种Provider可以配置多个模型：

### 1. GeminiProvider (type: gemini)
- 使用Google Gemini官方SDK
- 支持的模型：gemini-flash, gemini-pro, gemini-preview
- 需要代理访问

### 2. VectorEngineProvider (type: vectorengine)
- 使用OpenAI兼容SDK
- 支持的模型：deepseek-v3, qwen-plus, 以及其他OpenAI兼容模型
- 通过中转API访问，无需代理

## 当前配置 (config/llm_providers.yaml)

```yaml
providers:
  # Provider 1: Gemini官方
  gemini-official:
    type: gemini
    models:
      - alias: "gemini-flash"
        real_model: "gemini-flash-latest"
      - alias: "gemini-pro"
        real_model: "gemini-2.5-pro"
      - alias: "gemini-preview"
        real_model: "gemini-3.1-pro-preview"
  
  # Provider 2: VectorEngine中转
  vectorengine-relay:
    type: vectorengine
    base_url: "${VECTORENGINE_BASE_URL}"
    models:
      - alias: "deepseek-v3"
        real_model: "deepseek-v3.2"
      - alias: "qwen-plus"
        real_model: "qwen3.6-plus"

# 任务类型默认模型
task_defaults:
  longform: "deepseek-v3"      # 长文翻译 → VectorEngine
  post: "gemini-flash"         # 短文翻译 → Gemini
  analysis: "gemini-pro"       # 分析任务 → Gemini
  title: "gemini-flash"        # 标题翻译 → Gemini
  metadata: "gemini-flash"     # 元数据翻译 → Gemini
```

## 四步翻译法的模型使用

### 当你选择 `deepseek-v3` 作为默认模型时

**所有步骤都使用同一个模型**（deepseek-v3），通过VectorEngineProvider调用：

| 步骤 | 模型 | Provider | Temperature | 说明 |
|------|------|----------|-------------|------|
| Step 0 预扫描 | deepseek-v3 | VectorEngine | 0.3 | 术语提取 |
| Step 1 理解 | deepseek-v3 | VectorEngine | 0.7 | 文章分析 |
| Step 2 初译 | deepseek-v3 | VectorEngine | 0.5 | 批量翻译 |
| Step 3 反思 | deepseek-v3 | VectorEngine | 0.7 | 质量审查 |
| Step 4 润色 | deepseek-v3 | VectorEngine | 0.3 | 问题修复 |
| Step 5 风格润色 | deepseek-v3 | VectorEngine | 0.3 | 风格打磨 |

### 当你选择 `gemini-pro` 作为默认模型时

**所有步骤都使用同一个模型**（gemini-pro），通过GeminiProvider调用：

| 步骤 | 模型 | Provider | Temperature | 说明 |
|------|------|----------|-------------|------|
| Step 0 预扫描 | gemini-pro | Gemini | 0.3 | 术语提取 |
| Step 1 理解 | gemini-pro | Gemini | 0.7 | 文章分析 |
| Step 2 初译 | gemini-pro | Gemini | 0.5 | 批量翻译 |
| Step 3 反思 | gemini-pro | Gemini | 0.7 | 质量审查 |
| Step 4 润色 | gemini-pro | Gemini | 0.3 | 问题修复 |
| Step 5 风格润色 | gemini-pro | Gemini | 0.3 | 风格打磨 |

## 关键点

### ✅ 模型统一性
- **整个翻译流程使用同一个模型**
- 模型通过 `task_defaults.longform` 配置
- 默认是 `deepseek-v3`（VectorEngine中转）

### ✅ Provider自动选择
- 系统根据模型别名自动选择正确的Provider
- `deepseek-v3` → 自动使用 VectorEngineProvider
- `gemini-pro` → 自动使用 GeminiProvider

### ✅ Temperature分步优化
- 各步骤的temperature是硬编码的，针对不同任务特点优化
- 分析类任务（理解、反思）：0.7（需要创造性）
- 翻译类任务（初译）：0.5（平衡准确性和流畅性）
- 修复类任务（润色、风格）：0.3（需要精确控制）
- 提取类任务（预扫描）：0.3（需要稳定输出）

### ✅ 故障转移机制
```yaml
fallback_rules:
  within_provider:
    enabled: true
    strategy: "key_first"  # 先切换API Key，再切换模型
  
  cross_provider:
    enabled: true  # 跨Provider故障转移
    # gemini-official(优先级1) → vectorengine-relay(优先级2)
```

## VectorEngine的优势

### 为什么默认使用deepseek-v3？

1. **高性价比**
   - 成本远低于Gemini Pro
   - 质量接近GPT-4级别

2. **高并发**
   - max_concurrent: 100（Gemini只有2）
   - requests_per_minute: 1000（Gemini只有60）

3. **无需代理**
   - 直接访问，无需配置代理
   - 稳定性更好

4. **OpenAI兼容**
   - 标准的OpenAI API格式
   - 易于切换到其他模型（GPT-4, Claude等）

## 如何切换模型？

### 方法1: 修改配置文件

编辑 `config/llm_providers.yaml`:

```yaml
task_defaults:
  longform: "gemini-pro"  # 改为使用Gemini Pro
```

### 方法2: 环境变量

```bash
export LLM_MODEL_LONGFORM="gemini-pro"
```

### 方法3: 代码中指定

```python
from src.llm.factory import get_llm_provider_for_task

# 获取长文翻译的Provider（会使用task_defaults.longform配置）
provider = get_llm_provider_for_task("longform")
```

## 总结

### 回答你的问题

> "如果我选的模型是默认模型（中转的deepseek），怎么表现？"

**答案**：
1. ✅ **整个四步翻译流程都使用 deepseek-v3**
2. ✅ **通过 VectorEngineProvider 调用**（OpenAI兼容SDK）
3. ✅ **各步骤的temperature仍然是硬编码的**（0.3-0.7不等）
4. ✅ **不需要代理，直接访问中转API**
5. ✅ **支持100并发，速度快**
6. ✅ **成本低，适合大规模翻译**

### 与Gemini的区别

| 特性 | DeepSeek (VectorEngine) | Gemini (官方) |
|------|-------------------------|---------------|
| SDK | OpenAI兼容 | Google Gemini SDK |
| 代理 | 不需要 | 需要 |
| 并发 | 100 | 2 |
| 成本 | 低 | 中-高 |
| 配置 | 简单 | 需要代理配置 |
| 模型切换 | 容易（OpenAI兼容） | 仅限Gemini系列 |

**结论**：两种Provider的底层实现不同，但对于四步翻译法来说，**使用体验是一致的**，都是整个流程使用同一个模型，只是temperature根据步骤特点有所不同。
