## 1. 错误处理优化

**文件**: `src/api/middleware/error_handlers.py`

**改进内容**:
- 修改 `validation_exception_handler` 返回详细的字段验证错误
- 前端现在可以获取具体哪个字段出错及错误类型

**变更前**:
```python
return create_error_response(
    status_code=422,
    detail="Request validation failed",
    path=request.url.path,
)
```

**变更后**:
```python
return JSONResponse(
    status_code=422,
    content={
        "detail": "Request validation failed",
        "errors": error_details,  # 包含字段名、错误消息、错误类型
        "path": request.url.path,
    }
)
```

**影响**: 前端可以精确定位验证错误，改善用户体验

## 3. 慢请求监控

**文件**: `src/api/middleware/logging.py`

**改进内容**:
- 添加慢请求自动告警（阈值5秒）
- 超过阈值的请求会以 WARNING 级别记录

**实现**:
```python
SLOW_REQUEST_THRESHOLD = 5.0  # seconds

if process_time > SLOW_REQUEST_THRESHOLD:
    logger.warning(
        f"SLOW REQUEST: {request.method} {request.url.path} "
        f"took {process_time:.3f}s (threshold: {SLOW_REQUEST_THRESHOLD}s)"
    )
```

**影响**: 
- 自动识别性能问题
- 便于运维监控和性能优化

## 4. 环境变量加载验证

**文件**: 
- `src/api/app.py`
- `start_server.sh`

**改进内容**:

### 4.1 应用启动验证
在 `app.py` 中添加启动时环境变量检查：
```python
REQUIRED_ENV_VARS = ["VECTORENGINE_API_KEY", "VECTORENGINE_BASE_URL"]
missing_vars = [var for var in REQUIRED_ENV_VARS if not os.getenv(var)]

if missing_vars:
    error_msg = f"Missing required environment variables: {', '.join(missing_vars)}"
    logger.error(error_msg)
    raise RuntimeError(error_msg)
```

### 4.2 启动脚本验证
在 `start_server.sh` 中添加环境变量验证：
```bash
REQUIRED_VARS=("VECTORENGINE_API_KEY" "VECTORENGINE_BASE_URL")
MISSING_VARS=()

for var in "${REQUIRED_VARS[@]}"; do
  if [ -z "${!var}" ]; then
    MISSING_VARS+=("$var")
  fi
done

if [ ${#MISSING_VARS[@]} -gt 0 ]; then
  echo "ERROR: Missing required environment variables:"
  printf '  - %s\n' "${MISSING_VARS[@]}"
  exit 1
fi
```

**影响**: 
- 启动失败时立即报错，而非运行时才发现
- 明确指出缺失的环境变量
- 避免之前遇到的 "环境变量未加载导致 502 错误" 问题

## 5. LLM 故障转移监控

**文件**: `src/llm/provider_adapter.py`

**改进内容**:
- 添加故障转移统计指标
- 记录请求耗时和故障转移触发情况

**新增统计**:
```python
_fallback_stats = {
    "total_requests": 0,      # 总请求数
    "fallback_triggered": 0,  # 触发故障转移次数
    "total_attempts": 0,      # 总尝试次数
    "failed_requests": 0,     # 完全失败的请求数
}
```

**日志增强**:
```python
# 首次成功
logger.info(f"Success on first attempt in {duration:.2f}s")

# 故障转移成功
logger.warning(
    f"Fallback succeeded on attempt {idx}/{len(self.attempt_plan)} "
    f"after {duration:.2f}s"
)

# 完全失败
logger.error(
    f"All {len(self.attempt_plan)} attempts failed "
    f"after {duration:.2f}s. Stats: {_fallback_stats}"
)
```

**影响**: 
- 可量化故障转移效果
- 便于分析 Provider 稳定性
- 帮助优化故障转移策略
