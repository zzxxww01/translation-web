# 项目删除 UI 优化设计

## 设计理念：渐进式危险提示

将危险信号从按钮转移到确认对话框，通过**渐进式披露**的方式，让界面在平时保持克制，只在用户真正需要警告时才显示危险色。

---

## 优化前后对比

### 优化前
- ❌ 删除按钮使用 `variant="destructive"`，红色背景过于突兀
- ❌ 在未操作时就持续显示危险色，造成视觉噪音
- ❌ 确认对话框设计简陋，缺乏信息层次

### 优化后
- ✅ 删除按钮使用 `variant="ghost"`，平时低调融入界面
- ✅ hover 时才显示红色，提供即时反馈
- ✅ 确认对话框重新设计，信息层次清晰

---

## 设计细节

### 1. 删除按钮（克制状态）

```tsx
<Button
  variant="ghost"
  size="sm"
  onClick={handleDeleteClick}
  disabled={!currentProject || deleteProject.isPending}
  title="删除当前项目"
  className="group relative px-2.5 text-muted-foreground transition-all hover:text-destructive hover:bg-destructive/5 disabled:opacity-40"
>
  <Trash2 className="h-3.5 w-3.5 transition-transform group-hover:scale-110" />
</Button>
```

**设计要点**：
- **平时状态**：`text-muted-foreground` 灰色，低调不抢眼
- **hover 状态**：`hover:text-destructive hover:bg-destructive/5` 红色文字 + 淡红背景
- **微交互**：`group-hover:scale-110` 图标放大 10%，提供触觉反馈
- **禁用状态**：`disabled:opacity-40` 半透明，清晰传达不可用

### 2. 确认对话框（信息层次）

```tsx
<AlertDialogContent className="max-w-md">
  <AlertDialogHeader>
    <AlertDialogTitle className="flex items-center gap-2 text-lg">
      <div className="flex h-10 w-10 items-center justify-center rounded-full bg-destructive/10">
        <Trash2 className="h-5 w-5 text-destructive" />
      </div>
      确认删除项目
    </AlertDialogTitle>
    <AlertDialogDescription className="space-y-3 pt-2">
      <p className="text-sm leading-relaxed">
        确定要删除项目 <span className="font-semibold text-foreground">"{projectToDelete?.title}"</span> 吗？
      </p>
      <div className="rounded-md border border-destructive/20 bg-destructive/5 p-3">
        <p className="text-xs text-destructive-foreground/80">
          ⚠️ 此操作无法撤销，将永久删除：
        </p>
        <ul className="mt-2 space-y-1 text-xs text-destructive-foreground/70">
          <li>• 所有章节和段落</li>
          <li>• 翻译内容和进度</li>
          <li>• 项目术语表</li>
          <li>• 导出文件和缓存</li>
        </ul>
      </div>
    </AlertDialogDescription>
  </AlertDialogHeader>
</AlertDialogContent>
```

**设计要点**：
- **视觉锚点**：圆形图标容器 `bg-destructive/10`，建立视觉焦点
- **项目名高亮**：`font-semibold text-foreground` 让用户确认删除对象
- **警告卡片**：`border-destructive/20 bg-destructive/5` 淡红色背景，承载危险信息
- **信息列表**：详细列出删除范围，帮助用户理解后果
- **emoji 图标**：⚠️ 增强警告感知，无需额外图标库

---

## 设计原则

### 1. 渐进式披露（Progressive Disclosure）
不在用户未操作时就显示所有警告信息，而是通过交互逐步揭示：
- **第一层**：按钮平时灰色，融入界面
- **第二层**：hover 时显示红色，提示危险操作
- **第三层**：点击后对话框详细说明后果

### 2. 视觉层次（Visual Hierarchy）
确认对话框的信息组织：
1. **标题 + 图标**：建立视觉焦点
2. **项目名**：加粗高亮，确认删除对象
3. **警告卡片**：淡红背景，承载关键信息
4. **详细列表**：说明删除范围

### 3. 克制的色彩使用
- **平时**：灰色（`muted-foreground`）
- **hover**：红色文字 + 5% 透明度红色背景
- **对话框**：10-20% 透明度红色背景，避免过于刺眼

### 关键 CSS 类
```css
/* 按钮状态 */
text-muted-foreground          /* 平时灰色 */
hover:text-destructive         /* hover 红色文字 */
hover:bg-destructive/5         /* hover 淡红背景 */
disabled:opacity-40            /* 禁用半透明 */

/* 图标动画 */
transition-transform           /* 平滑过渡 */
group-hover:scale-110          /* hover 放大 */

/* 对话框警告卡片 */
border-destructive/20          /* 20% 透明度红色边框 */
bg-destructive/5               /* 5% 透明度红色背景 */
```

## 用户体验提升

1. **减少视觉噪音**：平时不显示红色，界面更清爽
2. **即时反馈**：hover 时红色 + 图标放大，提供明确反馈
3. **信息充分**：对话框详细列出删除范围，避免误操作
4. **操作流畅**：微动画提升交互质感

---

## 设计灵感

参考了现代设计系统的最佳实践：
- **Linear**：危险操作按钮平时低调，hover 时显示警告色
- **Notion**：确认对话框使用淡色背景承载警告信息
- **GitHub**：删除操作通过多层确认，逐步披露危险性

---

## 总结

通过**渐进式危险提示**的设计策略，将警告信号从按钮转移到确认对话框，让界面在平时保持克制优雅，只在用户真正需要时才显示危险色。这种设计既减少了视觉噪音，又保证了操作安全性。
