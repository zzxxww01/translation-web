整个翻译过程中，**模型是可配置的**，但各个步骤的**temperature参数有硬编码**。

## Temperature设计理念

### 高Temperature (0.5-0.7)
- **Step 1 理解**: 0.7 - 需要创造性分析
- **Step 2 初译**: 0.5 - 平衡准确性和流畅性
- **Step 3 反思**: 0.7 - 需要批判性思维

### 中Temperature (0.3-0.4)
- **Step 0 预扫描**: 0.3 - 稳定的术语提取
- **Step 4 润色**: 0.3 - 精确修复
- **Step 5 风格润色**: 0.3 - 精确打磨
- **标题翻译**: 0.3 - 稳定输出
- **重译**: 0.4 - 略微灵活

### 低Temperature (0.1-0.2)
- **格式修复**: 0.1 - 最保守，确保格式正确
- **元数据翻译**: 0.2 - 高度准确性

## 硬编码问题

### 当前状态
✅ **模型**: 完全可配置
- 通过环境变量 `GEMINI_MODEL` 全局配置
- 通过构造函数参数配置
- 特定步骤可通过 `model` 参数覆盖（如prescan使用flash）

❌ **Temperature**: 硬编码在各个方法中
- 无法通过配置文件调整
- 无法针对不同场景优化
- 需要修改代码才能调整

### 潜在改进方向

如果需要让temperature可配置，可以：

1. **在配置文件中定义**:
```yaml
llm:
  gemini:
    temperatures:
      prescan: 0.3
      understand: 0.7
      translate: 0.5
      reflect: 0.7
      refine: 0.3
      style_polish: 0.3
```

2. **在Provider初始化时读取**:
```python
def __init__(self, ...):
    self.temperatures = {
        "prescan": settings.gemini_temperature_prescan or 0.3,
        "translate": settings.gemini_temperature_translate or 0.5,
        # ...
    }
```

3. **在各个方法中使用**:
```python
def translate_section(self, ...):
    response = self.generate(
        prompt, 
        response_format="json", 
        temperature=self.temperatures.get("translate", 0.5)
    )
```
