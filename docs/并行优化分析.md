# 翻译系统并行优化分析

## 当前架构

### 翻译流程（串行）
```
1. 深度分析全文 (1次API调用)
2. 翻译标题和元信息 (1次API调用)
3. 翻译章节标题 (N次API调用，N=章节数)
4. 翻译来源说明 (1次API调用)
5. 逐章节翻译 (串行处理每个章节)
   - 章节预扫描 (1次API调用/章节)
   - 章节翻译 (多次API调用/章节)
     * 理解章节 (1次)
     * 批量初译 (1次或多次，取决于段落数)
     * 批量反思 (1次)
     * 批量润色 (1次，如果需要)
```

### 当前瓶颈
1. **章节串行处理**：179段落分布在多个章节中，每个章节必须等待前一个章节完成
2. **章节内步骤串行**：理解→初译→反思→润色，必须按顺序执行
3. **API并发限制**：当前配置未充分利用API的并发能力

## 可并行优化的地方

### 🔴 高优先级（影响最大）

#### 1. 章节级并行翻译
**位置**: `src/services/batch_translation_service.py:911` (for循环)

**当前代码**:
```python
for i, section in enumerate(project.sections):
    # 串行处理每个章节
    result = self.translator.translate_section(...)
```

**优化方案**:
```python
# 使用 asyncio.gather 并行翻译多个章节
async def translate_sections_parallel(sections, max_concurrent=3):
    semaphore = asyncio.Semaphore(max_concurrent)
    
    async def translate_with_limit(section, index):
        async with semaphore:
            return await asyncio.to_thread(
                self.translator.translate_section,
                section, all_sections, on_progress, project_id
            )
    
    tasks = [translate_with_limit(s, i) for i, s in enumerate(sections)]
    return await asyncio.gather(*tasks)
```

**预期提升**: 3-5倍速度提升（取决于并发数）

#### 2. 批量翻译内的段落并行
**位置**: `src/agents/four_step_translator.py:445` (_translate_batch)

**当前**: 一次API调用翻译整批段落（已经是批量的，但批次之间串行）

**优化方案**: 如果章节很长被分成多批，可以并行处理多批
```python
# 长章节分批翻译时，并行处理多批
if len(section.paragraphs) > self.paragraph_threshold:
    batches = self._split_into_batches(section.paragraphs)
    
    # 并行翻译多批（限制并发数）
    async def translate_batch_async(batch, idx):
        return await asyncio.to_thread(
            self._translate_batch,
            section, batch, understanding, all_sections, idx
        )
    
    batch_tasks = [translate_batch_async(b, i) for i, b in enumerate(batches)]
    batch_results = await asyncio.gather(*batch_tasks)
    translation_outputs = [item for batch in batch_results for item in batch]
```

**预期提升**: 对长章节有2-3倍提升

## 风险和注意事项

1. **术语一致性**: 并行翻译可能导致术语冲突增加，需要加强冲突检测
2. **API限流**: 过高并发可能触发API限流，需要实现退避策略
3. **内存占用**: 并行处理会增加内存占用，需要监控
4. **错误处理**: 并行任务的错误处理更复杂，需要仔细设计
