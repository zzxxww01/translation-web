## Task
Analyze the article and return structured context for downstream translation, critique, and revision.

1. Identify the single-sentence core theme.
2. Summarize the main argument flow in 3-6 key arguments.
3. Explain the article structure in a way that helps translators keep the logic intact.
4. Extract only the terminology that truly matters for consistency, reader comprehension, or analyst tone. Do not dump generic words.
5. Describe the source style and the Chinese voice that should preserve it.
6. List the main translation risks that could cause mistranslation, diluted judgment, or unreadable Chinese.
7. Produce 4-7 translation guidelines that the rest of the chain can directly follow. At least one guideline should address sentence restructuring and Chinese-native paragraph flow.
8. For GPU/semiconductor/CUDA-heavy articles, explicitly flag domain-specific terms that are commonly mistranslated (e.g., kernel vs operator, latent variable vs latent, floorplan vs layout, die shot, swizzle, bytes-in-flight) and include them in the terminology list with correct translations and rationale. Also ensure coverage of:
   - Architecture codenames (SM100, SM90, SM80) — annotate with architecture name (e.g., "SM100 为 Blackwell 架构代号")
   - Microarchitecture units (TPC, GPC, SM, MIO, MSHR) — include full English name and brief Chinese explanation
   - Framework-specific abstractions (CuTe, CUTLASS atom, Epilogue in GEMM context) — annotate with library context
   - SASS instruction mnemonics (LDGSTS, UBLKCP, UTMALDG) — include operation description
   - Profiling/debugging tools (NCU = Nsight Compute, NSight Systems) — include full product name
   All such terms should have `first_occurrence_note: true`.
9. Identify metaphors, analogies, and cultural references that may be opaque to Chinese readers. For each, note the reference, its cultural context, and whether a translator note is needed or a Chinese cultural equivalent can substitute.
10. Identify the article's domain mix (e.g., GPU + finance + cloud computing). For cross-domain articles, list terminology that has different translations in different domains, and specify which domain convention to follow in this article's context. Common cross-domain pitfalls: utilization rate (cloud) vs 出租率 (real estate); capacity vs 算力; financial modeling (investment) vs 财务建模 (corporate); token (AI) vs 代币 (blockchain).
11. Identify passages with strong rhetorical tone (sarcasm, emphasis, dramatic effect, irony) and note the intended emotional register so translators can match intensity rather than flatten it to academic prose.

## Rules
- Keep terminology selective. Do not dump generic words.
- Use `translation: null` only when the term should stay in English.
- `first_occurrence_note` should be true only when a short first-mention annotation materially helps Chinese readers.
- `guidelines` must be concrete enough to drive translation, critique, and revision.
- The article title translation must preserve all semantic components of the original title. Do not drop subject phrases (e.g., "Dissecting Nvidia Blackwell") even if the subtitle seems more keyword-rich.
- Do not wrap the JSON in Markdown.
