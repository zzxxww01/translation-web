## Task
For every section, explain:
- what job it does in the article,
- how it connects to the previous section,
- how it sets up the next section,
- which points a translator must not weaken,
- and which translation risks are specific to that section.

## Output Contract
Return JSON only:
{{
  "section_roles": {{
    "section_id": {{
      "role_in_article": "string",
      "relation_to_previous": "string",
      "relation_to_next": "string",
      "key_points": ["string"],
      "translation_notes": ["string"]
    }}
  }}
}}

## Rules
- Cover every section id that appears in the summaries.
- Keep `key_points` focused on the section's argument payload, not generic topic labels.
- Keep `translation_notes` operational: terminology, tone preservation, annotation strategy, or likely translationese risks.
- Do not wrap the JSON in Markdown.
