One of the most interesting questions in this debate: GPT-5.2 High has been benchmark-maxxing for a year and held pole position for the past two years since ChatGPT’s rise. The SWE-bench verified result is effectively the same, with GPT-5.2 high MMLU-pro and AIME 2025 in favor of 5.2, yet everyone is obsessed with Opus. So why is Claude Code dominating?

The answer is token efficiency. Opus 4.5 achieves dramatically better token efficiency than GPT-5.2 High, and this matters enormously for long-horizon task achievement.

![](https://substackcdn.com/image/fetch/$s_!huRl!,w_1456,c_limit,f_auto,q_auto:good,fl_progressive:steep/https%3A%2F%2Fsubstack-post-media.s3.amazonaws.com%2Fpublic%2Fimages%2Ff13013ab-dc6a-4355-9261-68aa478cf6ea_2025x1453.png)

Source: Artificial Analysis, [SemiAnalysis Tokenomics Team](https://semianalysis.com/tokenomics-model/), Generated by Claude Code

Long-horizon performance is fundamentally about compounding signal over multiple steps across context windows. Even though ChatGPT has a much larger context window, the efficiency curve of tasks-to-tokens tells a different story. Benchmark-maxxed ChatGPT may score higher on evals, but it takes an order of magnitude more tokens to accomplish tasks, which effectively ruins any long-horizon planning.

We think this is a deliberate choice Anthropic has made for some time. It’s not about spamming the most tokens (which creates some signal alongside a lot of noise) but about having the cleanest signal-to-noise ratio. This really matters over multi-context-window planning, where any compounding error leads to long-horizon task failure.

![](https://substackcdn.com/image/fetch/$s_!Y25K!,w_1456,c_limit,f_auto,q_auto:good,fl_progressive:steep/https%3A%2F%2Fsubstack-post-media.s3.amazonaws.com%2Fpublic%2Fimages%2Fc7b2dbfa-6b85-4d08-b665-1fdeaebe9b95_2105x1299.png)

Source: [SemiAnalysis Tokenomics Team](https://semianalysis.com/tokenomics-model/), generated by Claude Code

If you need 90% performance on a single context window, compounded over X steps, even small error rates guarantee failure at the end.

![](https://substackcdn.com/image/fetch/$s_!W7M1!,w_1456,c_limit,f_auto,q_auto:good,fl_progressive:steep/https%3A%2F%2Fsubstack-post-media.s3.amazonaws.com%2Fpublic%2Fimages%2F7da088ff-4352-4681-9baa-df566ac3dbee_787x443.png)

Source: Weka

Anthropic clearly values token efficiency. The new Sonnet 5 is allegedly coming in at a much smaller model size with a larger context window at similar performance to Opus 4.5. This effectively pushes the absolute max of the efficient token curve while expanding the context window. The capability difference in realized performance for coding agents will be ginormous. This is the gap Anthropic has over OpenAI.

OpenAI will respond. They must. A new pretrain is a prerequisite, as higher scale seems to improve longer context tasks. What made OpenAI’s models so powerful last year was having an extremely long chain-of-thought token context to achieve tasks. But over multi-context-window problems, this goes from a boon to a crutch because it compounds noise. The fix is straightforward: a new pretrain. We expect one in the next few months, but until then, Anthropic leads. The real adoption of OpenAI’s codex is much lower than first blush.
