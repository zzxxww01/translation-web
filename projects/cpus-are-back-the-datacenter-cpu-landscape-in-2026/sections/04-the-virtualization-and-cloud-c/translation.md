下一个重大转折点出现在 2000 年代末，即云计算的兴起。这也是整个 2010 年代数据中心 CPU 销量增长的核心驱动力。正如如今 GPU 新锐云厂商的运营模式一样，随着客户将资本支出转化为运营支出（opex），计算资源开始向公有云提供商以及诸如 AWS 等超大规模云服务商集中。受经济大衰退的余波冲击，许多企业已无力购买并维护自有服务器来运行其软件和服务。

云计算提供了一种更易于接受的“按量付费”商业模式。通过租用计算实例并在第三方硬件上运行工作负载，客户支出可随使用量变化而动态调整。这种可扩展性远优于自行采购服务器，因为自有服务器必须时刻保持满负荷运转，才能实现投资回报率 (ROI) 最大化。云技术还催生了更高效便捷的服务，例如 AWS Lambda 等提供的无服务器计算。这类服务能自动为软件分配计算资源，客户在运行特定任务前，无需再费心决定启动多少个实例。由于云厂商在后台包揽了几乎所有工作，计算能力彻底变成了一种商品。

构建安全且资源高效的云环境，其核心基石在于 CPU 硬件虚拟化。本质上，虚拟化允许单个 CPU 运行多个独立且安全的虚拟机 (VM) 实例，这些实例由 VMware ESXi 等虚拟机管理程序进行统一调度。多核 CPU 可被灵活划分，将每个虚拟机分配至单个核心或逻辑线程。虚拟机管理程序还能通过网络将实例迁移至不同的核心、插槽或服务器上，以此优化 CPU 利用率，同时确保数据与指令安全，免受同一 CPU 上其他运行实例的干扰。

云计算对虚拟化的需求，叠加 CPU 设计者为提升性能而引入的 SMT 技术，最终在 2018 年被幽灵 (漏洞) 与熔断 (漏洞) 所利用。当两个实例在同一物理核心的不同线程上运行时，攻击者可利用 CPU 核心的分支预测功能，窥探并拼凑出另一线程的数据。分支预测本是一项性能提升技术，通过在程序运行前提前猜测、提取并执行指令，使 CPU 保持满载状态。面对云端安全可能沦陷的危机，各家提供商紧急禁用 SMT，以彻底切断这一攻击路径。

尽管后续推出了补丁与硬件修复方案，但禁用 SMT 导致了高达 30% 的性能损失。这一代价成为英特尔挥之不去的阴影，并直接导致其在后续作出了一系列不合时宜的设计决策，我们将在下文对此展开详述。
