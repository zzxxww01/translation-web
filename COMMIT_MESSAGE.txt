## Security Fixes (9 issues)

### High Priority (3)
- Fix path traversal vulnerability in all paragraph endpoints
  - Add validate_path_component() to 7 endpoints in projects_paragraphs.py
  - Prevent arbitrary file access via ../../ patterns

- Fix SSE resource leak in four-step translation
  - Add proper cleanup in finally block with thread lock protection
  - Release translation slot on exit
  - Prevent memory leak from global dictionaries

- Fix race condition in global state management
  - Add threading.Lock() to protect _pending_conflict_events
  - Add threading.Lock() to protect _conflict_resolutions
  - Fix 5 concurrent access points in translate_projects.py

### Medium Priority (3)
- Prevent error message information disclosure
  - Hide stack traces in production (DEBUG=false)
  - Log detailed errors server-side only
  - Return generic error messages to clients

- Implement request body size limit
  - Add 100MB limit in LimitUploadSize middleware
  - Return 413 status code for oversized requests
  - Prevent OOM attacks

- Add file write retry mechanism for Windows
  - Retry 3 times with exponential backoff (0.1s, 0.2s, 0.3s)
  - Handle file lock conflicts on Windows
  - Log detailed error messages

### Low Priority (3)
- Add CORS configuration
  - Support whitelist via CORS_ORIGINS env var
  - Default to localhost for development

- Add automated backup scripts
  - Linux: scripts/backup.sh with cron support
  - Windows: scripts/backup.bat with Task Scheduler support
  - Support cloud storage (S3/OSS)

- Clean sensitive information from .env.example
  - Remove real API endpoints
  - Comment out proxy configuration
  - Add CORS configuration example
