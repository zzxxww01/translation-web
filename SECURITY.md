## 报告漏洞

如果您发现安全漏洞，请通过以下方式报告：

1. **私密报告**：请发送邮件至项目维护者（不要公开披露）
2. **包含信息**：
   - 漏洞描述
   - 复现步骤
   - 影响范围
   - 建议修复方案（可选）

我们会在 **48 小时内**响应您的报告，并在 **7 天内**提供修复计划。

## 安全最佳实践

### 部署建议

1. **网络隔离**
   - 部署在内网或 VPN 环境
   - 使用反向代理（Nginx/Caddy）并配置 HTTPS
   - 配置防火墙规则限制访问

2. **环境变量**
   - 不要将 `.env` 文件提交到版本控制
   - 使用强密码和定期轮换 API 密钥
   - 限制环境变量文件的读取权限（chmod 600）

3. **数据备份**
   - 定期备份 `projects/` 目录
   - 使用 `scripts/backup.sh` 或 `scripts/backup.bat`
   - 将备份存储到安全的离线位置

4. **监控**
   - 监控 `/api/health` 端点
   - 设置磁盘空间告警（< 10GB）
   - 监控内存使用（> 80% 告警）
   - 定期检查日志文件

### 配置建议

```bash
# .env 示例（生产环境）
DEBUG=false
CORS_ORIGINS=https://your-domain.com
API_HOST=127.0.0.1  # 仅监听本地，通过反向代理暴露
API_PORT=54321
LOG_LEVEL=WARNING
```
